## (e) Prompt Caching

**NOT USED.** Zero instances of `cache_control`, `ephemeral`, or any prompt caching parameter in the `messages.create()` call (line 5158). The system prompt (including the ~15K char rulebook) is sent in full on every Sonnet/Opus call.

**Impact:** Adding `cache_control: {"type": "ephemeral"}` to the rulebook block in the system prompt would cache it across calls. With Opus at $15/M input, a ~15K char rulebook (~4K tokens) costs ~$0.06/call. With caching (90% discount on cache hits), that drops to ~$0.006/call. Over 10 calls/day, saves ~$0.54/day.

## Implementation Plan

### 1. Prompt caching (biggest win, affects /deep AND /cio)

Change the `messages.create()` call to use structured system prompt with cache_control:

```python
# Before:
response = await claude_client.messages.create(
    model=model, max_tokens=MAX_TOKENS_PER_REPLY,
    system=dated_system, tools=TOOLS, messages=messages,
)

# After:
system_blocks = [{"type": "text", "text": dated_system}]
if model != CLAUDE_MODEL_HAIKU and _RULEBOOK_TEXT:
    system_blocks.append({
        "type": "text",
        "text": _RULEBOOK_TEXT,
        "cache_control": {"type": "ephemeral"},
    })
response = await claude_client.messages.create(
    model=model, max_tokens=MAX_TOKENS_PER_REPLY,
    system=system_blocks, tools=TOOLS, messages=messages,
)
```
