## Race condition guard

1. Terminal state block: FILLED/REJECTED/CANCELLED orders reject further transitions
2. Monotonic check: logs warning on invalid transitions (e.g., SENT→STAGED) but allows for robustness
3. Status history JSON preserves full timeline for forensic analysis
