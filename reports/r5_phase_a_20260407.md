For IBKR events that arrive with no matching pending_orders row.

```sql
CREATE TABLE IF NOT EXISTS orphan_order_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    event_type TEXT NOT NULL,
    ib_order_id INTEGER,
    ib_perm_id INTEGER,
    status TEXT,
    payload TEXT,
    received_at TEXT NOT NULL DEFAULT (datetime('now'))
)
```

## New module: agt_equities/order_state.py

- `OrderStatus` enum: 14 states (STAGED through DUPLICATE_SKIPPED)
- `VALID_TRANSITIONS`: directed graph of allowed transitions
- `TERMINAL_STATES`: FILLED, REJECTED, CANCELLED, etc. (no further transitions)
- `IBKR_STATUS_MAP`: maps IBKR orderStatus strings to our enum
- `append_status()`: appends to status_history JSON, validates transitions, never crashes
- `backfill_status_history()`: migrates existing rows
